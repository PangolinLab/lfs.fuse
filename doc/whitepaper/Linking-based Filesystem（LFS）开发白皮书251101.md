📖 概述
LFS (Finking-based Filesystem) 是一种融合 版本化、去重存储、智能缓存 的混合文件系统。它通过六分区架构、差异化存储机制和智能优化策略，实现 安全、轻量、高效 的目标。LFS 不仅减少冗余存储，还能利用机器学习预测访问模式，提升整体性能与可靠性。

---

🗂 六分区架构与功能

- .source  
  存放原始文件数据，支持 AES 或 ChaCha20 加密。为避免单文件过大，.source 采用 分片存储，每个文件由多个分片组成，link 记录分片顺序。这样天然支持分块差异记录与分块拼合，提升大文件处理效率。  

- .index  
  存放每个源文件的合法映射列表，避免未授权的访问，类似元数据数据库。即使 .index 丢失，管理员仍可通过 link 中的冗余信息恢复 .source 与 .difference，保证系统的容错性。

- link  
  普通用户唯一可见的分区，存放快捷方式（映射），避免重复存储。用户通过 link 访问文件时，系统会自动拼合 .source 与 .difference，对用户透明。 link 还保存每个文件分片的哈希值和 source 分片顺序和对应的 difference，用于定期校验数据完整性。  

- .difference  
  存放不同映射的差异（类似 Git patch 或 COW delta）。差异按分片记录，避免全文件重写。若差异过大，则新建独立 .source；若多个 .source 相似度高，则合并为一个 .source，差异保留在 .difference。  

- .temp  
  临时拼合缓存区，用于快速合成“源文件 + 差异”。高频文件块会直接缓存到内存，低频数据则落盘存储。后台守护进程结合机器学习预测与 LRU/LFU 策略，动态清理 .temp，平衡 CPU 与 I/O 压力。  

- .ml  
  存放机器学习模型、日志与训练数据。模型可预测文件访问模式，不仅决定哪些文件提前拼合到 .temp，还可推荐合适的压缩算法（如文本用 ZSTD，高压缩比；视频用 LZ4，低延迟），实现 CPU 与 I/O 的平衡。  

---

⚙️ 配置机制（lfs.config.ini）

- 差异策略  
  - 独立模式：差异过大 → 新建 .source  
  - 合并模式：多个 .source 相似度高 → 合并为一个 .source，差异保留在 .difference  

- 缓存与清理  
  - 配置 .temp 的清理周期与策略（LRU、LFU、预测驱动）。  
  - 配置内存缓存大小，用于高频文件块。  

- 压缩与加密  
  - 按文件类型选择压缩算法。  
  - 所有分区均可选择加密，支持 per-partition key。  

---

🚀 关键特性与优势

- 存储节省：复制文件时不再写入完整数据，只生成映射与差异。  
- 高效访问：  
  - 高频文件块缓存到内存，低频数据落盘。  
  - 大文件支持分片并行读取，提升吞吐量。  
- 灵活性：差异策略可调，适应代码版本管理、大文件存储等不同场景。  
- 安全性：全分区可选加密，管理员权限隔离，支持零知识存储。  
- 可靠性：  
  - .index 非决定性，可通过 link 恢复。  
  - 事务化操作保证原子性。  
  - 定期哈希校验，防止 silent corruption。  
- 智能化：  
  - ML 预测访问模式，提前生成 .temp。  
  - 动态推荐压缩算法，平衡 CPU 与 I/O。
